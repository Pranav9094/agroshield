1. Project Overview & Problem Statement
What is AgroShield?
AgroShield is an AI-powered weed detection system built for precision agriculture. It uses a YOLOv8
object detection model trained on real farm images to identify weeds in a field photo, draw bounding boxes
around them, calculate spray zones, and give farmers targeted recommendations — instead of spraying
chemicals blindly across the whole field.
The Real Problem
Problem
Impact
Farmers spray chemicals across the entire field
80% of pesticide is wasted on weed-free areas
No way to identify weed-dense zones from crop zones
Higher cost, lower yield
Manual scouting is time-consuming and error-prone
Delays treatment, weeds spread further
Chemical overuse harms soil health
Long-term productivity loss
Our Solution — In One Line
AgroShield detects weeds using AI, shows where they are, and tells you exactly
where to spray.
Upload a field image → AI detects weeds → Bounding boxes drawn → Spray coordinates generated →
Farmer saves chemicals, reduces cost, protects crops.
Impact Statement (use this in PPT)
 Reduce pesticide usage by 40–60% through targeted spraying
 Save Rs 2,000–5,000 per acre per season in chemical costs
 Scalable to drone integration and smart sprayer systems
 Supports India's Digital Agriculture Mission and precision farming goals

3. Tech Stack & Tools — Install Commands
M1 — AI Environment Setup
Run these commands on the GPU laptop:
# Step 1 — Python environment
python -m venv agroshield_env
source agroshield_env/bin/activate   # Windows: agroshield_env\Scripts\activate
# Step 2 — Install YOLOv8
pip install ultralytics opencv-python pillow
# Step 3 — Verify GPU (CUDA)
python -c "import torch; print(torch.cuda.is_available())"
# Expected output: True
# Step 4 — Download dataset (Roboflow WeedCrop)
# Go to: https://roboflow.com → search "WeedCrop" → Download YOLOv8 format
# OR use roboflow package:
pip install roboflow
# Then run the download script (see M1 prompts section)
M2 — Frontend Setup
# Step 1 — Create React project
npm create vite@latest agroshield-frontend -- --template react
cd agroshield-frontend
npm install
# Step 2 — Install dependencies
npm install axios
# Step 3 — Run dev server
npm run dev
# App runs at: http://localhost:5173
M3 — Backend Setup
# Step 1 — Create backend folder
mkdir agroshield-backend && cd agroshield-backend
python -m venv venv
source venv/bin/activate
# Step 2 — Install FastAPI
pip install fastapi uvicorn python-multipart pillow ultralytics
# Step 3 — Run server
uvicorn main:app --reload --host 0.0.0.0 --port 8000
# API runs at: http://localhost:8000
# Docs at:    http://localhost:8000/docs

14. Judge Q&A; Preparation
 Q1
What dataset did you use to train the model?
We used the WeedCrop dataset available on Roboflow — it contains thousands of real field images with
labelled bounding boxes for weeds and crops separately. This gave our model actual agricultural data
rather than synthetic images.
 Q2
What is your model accuracy?
Our model achieved [insert your mAP50 score] mean Average Precision on the validation set. For
judges: the goal of this prototype is not 99% accuracy — it's demonstrating that AI-based targeted
detection is feasible and scalable. Accuracy improves significantly with larger and more diverse
datasets.
 Q3
How is this different from just spraying the whole field?
Blanket spraying treats 100% of the field equally — wasting 60-80% of chemicals on weed-free areas.
AgroShield detects only weed-dense zones and generates precise spray coordinates. Studies in
precision agriculture show this reduces chemical use by 40-60%, directly cutting farmer costs.
 Q4
Can this work on a drone in real time?
YOLOv8 was specifically designed for real-time inference — it runs at 30+ FPS on edge hardware like
NVIDIA Jetson. Our current prototype is image-based, but the detection pipeline is identical to what a
drone camera would use. Drone integration is our Phase 2 scope.
 Q5
What if weeds look similar to crops?
This is exactly why we need a trained AI model instead of color filters. YOLOv8 learns texture, shape,
and context from thousands of examples. We can further improve with more training data, including
similar-looking species. Domain-specific datasets for Indian crops like wheat and rice exist on Roboflow.
 Q6
Why FastAPI and not Django or Flask?
FastAPI is significantly faster than Flask for API workloads — it uses async under the hood. For image
processing with ML models, response time matters. FastAPI also auto-generates OpenAPI
documentation which made testing easier.
 Q7
What is the business model / who is the customer?

Three customer segments: individual farmers (mobile app subscription), agri-tech companies
integrating our API, and government agricultural programs deploying this at scale. India's PM-KISAN
and Digital Agriculture Mission both create demand for exactly this type of precision farming solution.
 Q8
Is the code on GitHub?
Yes — our repository is at github.com/[username]/agroshield. It includes the full source code, README
with setup instructions, and model training results. All five team members contributed through separate
feature branches with pull requests.
